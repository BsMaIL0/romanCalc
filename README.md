# 🏛️ Калькулятор римских чисел

Калькулятор арифметических выражений с римскими цифрами, записанных в обратной польской нотации (ОПН/постфиксной форме). Поддерживает базовые арифметические операции и проверку диапазона допустимых значений.

## 📥 Входные данные

Файл с выражениями в обратной польской нотации. В каждой строке — одно выражение, операнды и операторы разделены пробелами.

- **Римские числа**: от `I` до `MMMCMXCIX` (1–3999)
- **Операторы**: `+`, `-`, `*`, `/` (целочисленное деление)

## 📤 Выходные данные

Файл, в котором каждая строка исходного файла дополнена ` = ` и результатом вычисления (в римских цифрах) или `ERR`, если результат выходит за диапазон 1–3999.

## 📋 Пример

**Входной файл `input.txt`:**

```
V II + IV -
IV VI * VIII /
CC XX *
```

**Выходной файл `output.txt`:**

```
V II + IV - = III
IV VI * VIII / = III
CC XX * = ERR
```


### Пояснение

1. `V II + IV -` → 5 + 2 = 7, 7 - 4 = 3 → `III`
2. `IV VI * VIII /` → 4 * 6 = 24, 24 / 8 = 3 → `III`
3. `CC XX *` → 200 * 20 = 4000 → выходит за диапазон 1–3999 → `ERR`

## 🛠️ Алгоритм решения

### Конвертация римских чисел в арабские (`romanToArabic`)

Чтение строки слева направо с учётом правил вычитания:
- `IV` = 4, `IX` = 9
- `XL` = 40, `XC` = 90
- `CD` = 400, `CM` = 900

При обнаружении символа, который может участвовать в вычитании (`I`, `X`, `C`), проверяется следующий символ. Если следующий символ образует вычитаемую комбинацию — вычитаем, иначе — прибавляем.

### Конвертация арабских чисел в римские (`arabicToRoman`)

Используется табличное представление с убывающими значениями:
- Сначала все тысячи (`M`), затем 900 (`CM`), 500 (`D`), 400 (`CD`) и т.д.
- Для каждого значения из таблицы, пока число больше или равно ему, добавляем соответствующий римский символ и вычитаем значение.

### Вычисление RPN (обратной польской нотации)

Используется стек, реализованный на статическом массиве:

1. Чтение строки слева направо, разделение по пробелам
2. Для каждого токена:
   - **Операнд** (римское число): конвертируем в арабское и кладём в стек
   - **Оператор**: извлекаем два верхних числа из стека, выполняем операцию, результат кладём обратно в стек
3. После обработки всех токенов:
   - Если в стеке ровно один элемент и результат в диапазоне 1–3999 → конвертируем в римское и выводим
   - Иначе → выводим `ERR`

## 🔧 Реализация

### Стек
```
int stack[100];   // максимум 100 чисел по условию
int top = -1;     // вершина стека

// Push (положить число)
stack[++top] = value;

// Pop (извлечь два числа)
int b = stack[top--];
int a = stack[top--];
stack[++top] = result;  // результат обратно в стек
```

### Проверка оператора

```
bool isOperator(char c) {
    return c == '+' || c == '-' || c == '*' || c == '/';
}
```

## ⚠️ Ограничения

- **STL запрещён** — нельзя использовать `std::string`, `std::vector`, `std::stack` и т.д.
- В каждом выражении **не более 100 чисел**
- Римские числа только в диапазоне **1–3999**
- Поддерживаются только операторы `+`, `-`, `*`, `/` (целочисленное деление)

./romanCalc input.txt output.txt
